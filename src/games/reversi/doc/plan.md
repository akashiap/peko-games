# リバーシ Web ゲーム 実装プラン

## 概要
スマートフォンのブラウザで動作する、2人用 / 1人用（CPU対戦）のリバーシゲームを、素のHTML/CSS/JavaScriptで実装する。

## 技術スタック
- HTML5 / CSS3 / Vanilla JavaScript（フレームワーク不使用）
- 外部ライブラリ・画像・音声・Webフォントは使用しない（すべてCSS/SVGで描画）

## ファイル構成
```
/home/cmuser/game-01/
├── index.html      # 画面構造
├── style.css       # クラシック緑盤面・スマホ対応スタイル
├── script.js       # ゲームロジック + AI
└── doc/
    └── plan.md     # 本プラン
```

## 画面仕様
### タイトル画面
- モード選択：1人用 / 2人用
- 1人用選択時：難易度選択（弱 / 中 / 強）
- 開始ボタン

### 対戦画面
- 8×8 盤面
- 現在の手番表示（黒 / 白）
- 石数カウント（黒：N / 白：N）
- リセットボタン
- タイトルへ戻るボタン
- 合法手なし時は自動パス＋通知

### 終局画面
- 勝者表示（黒の勝ち / 白の勝ち / 引き分け）
- 最終石数
- もう一度プレイボタン
- タイトルへ戻るボタン

## コア機能（共通ロジック）
1. 盤面状態管理：8×8 の二次元配列（0=空, 1=黒, 2=白）
2. 合法手判定：8方向への挟み判定
3. 石を置く処理＋反転処理
4. 合法手のハイライト表示
5. パス処理（合法手なし時に自動パス、両者パスで終局）
6. 終局判定と勝敗カウント

## 1人用 AI 設計
2人用ロジックの「player2 の入力」を AI に差し替える構成にし、共通コードを最大化する。

| 難易度 | アルゴリズム |
|--------|--------------|
| 弱     | 合法手からランダム選択 |
| 中     | 位置評価テーブル（角=高得点、角隣=低得点、辺=中）による1手評価 |
| 強     | ミニマックス + αβ枝刈り（3〜4手先読み、スマホ性能考慮） |

- AI 思考中は約 500ms のディレイ＋「思考中...」表示で自然な体感を実現
- 重い処理時は UI スレッドをブロックしないよう `setTimeout` で分割

## スマホ対応
- `<meta name="viewport">` 設定
- 盤面サイズは `min(100vw, 100vh - UI高さ)` で自動調整
- タップ操作前提（誤タップ防止のため十分なマス幅を確保）
- 画面回転（縦横）に対応
- ピンチズーム抑制で誤操作を防止

## デザイン（クラシック）
- 盤面：深緑（#0a6b3b 系）
- 石：黒（#1a1a1a）／白（#f5f5f5）、わずかなグラデーションで立体感
- 枠線：黒、座標表示なし
- フォント：システムフォント（`-apple-system, sans-serif` 系）
- アニメーション：石を置いた際のフェードイン／反転を控えめに

## 著作権・商標への配慮
- 「オセロ」は登録商標のため、画面上の表記には **「リバーシ」** を使用する
- 画像・音声等の外部素材は使用しない（すべて CSS / SVG で描画）
- フォントはシステム標準のみ
- 第三者のコード・素材を流用しない

## 実装順序
1. HTML 骨組み（タイトル / 対戦 / 終局の3画面切替）
2. CSS（盤面・石・レイアウト・スマホ対応）
3. JS：盤面初期化と描画
4. JS：合法手判定＋反転処理
5. JS：手番進行＋パス＋終局判定
6. JS：2人用フローの完成
7. JS：AI（弱 → 中 → 強の順に実装）
8. 動作確認とスマホ表示調整
